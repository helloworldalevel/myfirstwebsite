📖 Part 1: Getting Started with HTML

What is HTML and why is it fundamental to web development?

🌐💻📝

What is HTML?

HTML (HyperText Markup Language): The standard language used to create web pages. HTML provides structure and content to web documents.

Key Facts About HTML:

  • Markup Language - Uses tags to describe content
  • Not Programming - HTML doesn't perform logic, it structures
  • Universal - Used by every website on the internet
  • Evolving - HTML5 is the current standard
  • Browser-Interpreted - Browsers display HTML content

💼 Real-World Use:

Every website you visit (Facebook, YouTube, Google, BBC News) uses HTML to structure its content. Browsers read the HTML code and display it as a beautiful webpage.

Why Do We Need HTML?

The Three Layers of Web Development:

Layer Purpose What It Does
HTML Structure Organizes content - headings, paragraphs, links, images
CSS Styling Controls appearance - colors, fonts, layout
JavaScript Behavior Adds interactivity - animations, form validation

🎯 Example:

A blog post page: HTML structures the title, paragraphs, and comments. CSS makes it look beautiful. JavaScript enables the comment submission form.

🔧 Part 2: The Building Blocks of HTML

Exploring Tags, Elements, and Attributes

🏗️🧩🔨

HTML Tags: The Foundation

HTML Tags: Keywords surrounded by angle brackets that tell browsers how to interpret content.

Tag Structure:

<tagname> content </tagname>

Two Types of Tags:

Container Tags Have opening & closing tags
<p>text</p>
Self-Closing Tags Single tag only
<img />

Examples of Tags:

<h1>, <p>, <div>, <span>, <a>, <img />, <br />

HTML Elements Explained

HTML Element: A complete unit consisting of an opening tag, content, and closing tag. The entire package working together.
Opening tag: <p> Content: This is a paragraph Closing tag: </p> Together = HTML Element

Key Distinction:

Tag vs Element Example
Tag - just the code <p> or </p>
Element - complete unit <p>Hello World</p>

HTML Attributes: Adding Properties

Attribute: Extra information added to HTML tags to modify their behavior or provide additional details.

Attribute Syntax:

<tag attribute="value">content</tag>

Common Attributes:

  • id - Unique identifier for an element
  • class - Classifies elements for styling
  • style - Inline CSS styling
  • alt - Alternative text for images
  • href - URL for links

Attributes in Action:

<img src="photo.jpg" alt="My Photo" />

The Basic HTML Document Structure

Every HTML page has this framework:

<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <h1>Welcome!</h1> </body> </html>

What Each Part Does:

  • <!DOCTYPE html> - Declares this is HTML5
  • <html> - Root element containing everything
  • <head> - Metadata, title, links (not visible on page)
  • <body> - All visible page content

🎯 Part 3: Global Attributes

Attributes that work on almost every HTML element

🔑📌💾

Global Attributes: Universal Properties

Global Attributes: Attributes available to all HTML elements that provide common functionality.

The Most Important Global Attributes:

id Unique identifier for an element. Must be unique within the page.
class Classifies elements for CSS styling and grouping.
style Inline CSS styling directly on the element.
lang Specifies the language of the element's content.
title Tooltip text shown on hover.
hidden Hides the element from display.

The id Attribute: Unique Identity

id Attribute: Gives a unique identifier to an element so it can be targeted by CSS or JavaScript.

Important Rules:

  • ID must be unique within the entire page
  • Can be used with CSS (#) and JavaScript
  • Each element can have only one id
<h1 id="main-title">Welcome to My Site</h1> /* CSS targeting this ID */ #main-title { color: blue; }

💼 Real-World Example:

Online forms use IDs to identify input fields: <input id="email-address" />

The class Attribute: Styling Groups

class Attribute: Classifies elements so multiple elements can share the same styling.

Key Differences from id:

Feature id class
Uniqueness One per page Reusable on many
Per element One id Multiple classes
CSS selector #name .name
<p class="highlight intro">Welcome</p> <p class="highlight">Important info</p> /* CSS targeting class */ .highlight { background: yellow; }

The style Attribute: Inline Styling

style Attribute: Applies CSS directly to an individual element inline.

When to Use Inline Styles:

  • Quick testing during development
  • One-time styling that won't repeat
  • Not recommended for production code
<p style="color: red; font-size: 18px;"> Red paragraph </p>

Example - Styling a Div:

<div style="background-color: #e8f4f8; padding: 20px;"> Styled content </div>

The lang Attribute: Language Declaration

lang Attribute: Specifies the language of the element's content. Important for accessibility and SEO.

Uses:

  • Helps screen readers pronounce words correctly
  • Assists search engines understand content
  • Enables browser spell-check features
<html lang="en"> ...page content... </html> <p lang="fr">Bonjour!</p>

Common Language Codes:

  • en = English | fr = French | de = German | es = Spanish

🏗️ Part 4: Structural HTML Elements

Building block elements that organize page content

📋📑📄

Heading Elements: Hierarchies

Heading Tags (h1-h6): Define headings and subheadings in order of importance. h1 is largest/most important.

The Six Heading Levels:

<h1>Main Page Heading</h1> <h2>Major Section</h2> <h3>Subsection</h3> <h4>Minor Heading</h4> <h5>Small Heading</h5> <h6>Smallest Heading</h6>

Best Practices:

  • Use only one h1 per page (the main title)
  • Follow logical order - don't skip levels
  • Use headings to create document outline

The Paragraph Element: <p>

<p> Tag: Represents a paragraph of text. Browsers automatically add spacing before and after paragraphs.
<p> This is a paragraph of text that explains something important. It can contain multiple sentences. </p> <p> This is another paragraph. Each paragraph is separate. </p>

Important Rules:

  • Paragraphs are block-level elements (full width)
  • Text within is automatically wrapped
  • Never nest paragraphs inside other paragraphs

❌ What NOT to do:

<p>Outer paragraph <p>Inner paragraph</p></p>

Emphasis Elements: <em> & <strong>

<em> and <strong>: Semantic tags that emphasize text. em = emphasis, strong = importance.

The Difference:

Tag Meaning Appearance
<em> Emphasis Italics
<strong> Strong importance Bold
<p> This word is <em>emphasized</em> and this word is <strong>important</strong>. </p>

When to Use:

  • <em> - Stress emphasis in speech
  • <strong> - Important warnings, labels, or strong importance

The Line Break Element: <br />

<br /> Tag: A self-closing element that creates a line break in text.

Key Facts:

  • Self-closing - no closing tag needed
  • Creates a single line break
  • Use for single breaks, not spacing
<p> Line 1<br /> Line 2<br /> Line 3 </p>

⚠️ When NOT to Use:

Don't use multiple <br /> tags to create spacing. Use CSS margins and padding instead!

🔗 Part 5: Links and Navigation

Creating connections between pages

🔗🌐🗂️

The Anchor Element: <a>

<a> Tag: Creates hyperlinks to other pages, files, or locations. Essential for navigation.

The href Attribute:

<a href="https://www.example.com"> Visit Example Site </a>

Types of Links:

External Links Link to other websites. Use full URL starting with http/https.
Internal Links Link to pages on same website. Use relative paths like "page.html".
Email Links Create mailto links: href="mailto:user@example.com"
Anchor Links Jump to section on page: href="#section-id"

Link Examples: Practical Patterns

External Link:

<a href="https://www.bbc.co.uk"> BBC News </a>

Internal Link:

<a href="about.html"> About Us </a>

Email Link:

<a href="mailto:contact@example.com"> Email Us </a>

Open in New Tab:

<a href="https://example.com" target="_blank"> External Site </a>

🖼️ Part 6: Adding Content

Images, forms, tables, and multimedia

🖼️📋📹

The Image Element: <img />

<img /> Tag: A self-closing element that embeds images in web pages.

Essential Attributes:

src Path to image file (required)
alt Alternative text if image fails (required for accessibility)
width Image width in pixels
height Image height in pixels
<img src="mountain.jpg" alt="Beautiful mountain landscape" width="400" height="300" />

Lists: Ordered and Unordered

List Elements: <ul> for unordered (bullets), <ol> for ordered (numbered), <li> for items.

Unordered List (<ul>):

<ul> <li>First item</li> <li>Second item</li> <li>Third item</li> </ul>

Ordered List (<ol>):

<ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol>

Tables: Organizing Data

Table Elements: <table>, <tr> (row), <td> (cell), <th> (header cell)
<table> <tr> <th>Name</th> <th>Age</th> </tr> <tr> <td>John</td> <td>25</td> </tr> </table>

Key Elements:

  • <table> - Container for entire table
  • <tr> - Table row
  • <th> - Table header cell (bold, centered)
  • <td> - Table data cell

Forms: Collecting User Input

Form Element: <form> contains input fields, buttons, and other interactive elements.
<form> <label for="name">Name:</label> <input type="text" id="name" /> <label for="email">Email:</label> <input type="email" id="email" /> <button type="submit">Submit</button> </form>

Common Input Types:

  • text - Single line text
  • email - Email validation
  • password - Hidden characters
  • checkbox - Multiple choices
  • radio - Single choice

📚 Part 7: Semantic HTML

Writing meaningful, accessible HTML

🏛️📖✨

Semantic HTML: What & Why

Semantic HTML: Using HTML tags that clearly describe their meaning and purpose, not just appearance.

Semantic vs Non-Semantic:

Non-Semantic Semantic Meaning
<div> <article> Main content article
<div> <nav> Navigation menu
<div> <header> Page header
<div> <footer> Page footer

Why It Matters:

  • Accessibility - Screen readers understand structure
  • SEO - Search engines rank semantic markup better
  • Maintainability - Code is easier to understand

Semantic Page Structure

A Well-Structured Page:

<body> <header>...logo and title...</header> <nav>...navigation menu...</nav> <main> <article>...main content...</article> <aside>...sidebar...</aside> </main> <footer>...copyright info...</footer> </body>

Key Semantic Elements:

  • <header> - Introductory content
  • <nav> - Navigation section
  • <main> - Main content (once per page)
  • <article> - Self-contained content
  • <section> - Thematic grouping
  • <aside> - Sidebar/related content
  • <footer> - Footer information

When to Use <div>

<div> Tag: A generic container when no semantic element fits. Use only when no better option exists.

Proper Use of <div>:

<!-- WRONG: Using div instead of article --> <div class="article"> ...content... </div> <!-- RIGHT: Using semantic tag --> <article> ...content... </article> <!-- OK: Using div for layout --> <div class="card-container"> ...three card elements... </div>

Rule of Thumb:

✓ Use <div> for layout wrappers when semantic element doesn't fit

✗ Don't use <div> when a semantic element would be more appropriate

✅ Part 8: HTML Best Practices

Writing clean, professional HTML code

🎯🧹📝

HTML Syntax Best Practices

Well-Organized HTML: Follows consistent style rules for readability and maintainability.

The Rules (from Spec 7.1.4):

  • Lowercase - All element names and attributes in lowercase
  • Indentation - Indent nested elements (2-4 spaces)
  • Double Quotes - Always use " not ' for attributes
  • No Values on Booleans - disabled, checked (not disabled="disabled")
  • Remove Slashes - Self-closing tags: <br> not <br />
<!-- CORRECT --> <form> <input type="checkbox" checked> <br> <input type="submit" value="Send"> </form>

Proper Indentation

Good Indentation Pattern:

<body> <header> <h1>Welcome</h1> </header> <main> <article> <h2>Article Title</h2> <p>First paragraph...</p> <p>Second paragraph...</p> </article> </main> </body>

Benefits:

  • Makes code easier to read
  • Shows nesting structure clearly
  • Helps spot missing closing tags

The <head> Element: Metadata

<head> Element: Contains metadata about the document - not visible on the page.
<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Page description for search engines"> <title>My Awesome Website</title> <link rel="stylesheet" href="styles.css"> <script src="script.js"></script> </head>

Essential Head Elements:

  • <meta charset> - Character encoding (UTF-8)
  • <meta viewport> - Mobile responsiveness
  • <title> - Page title (shown in browser tab)
  • <link> - Link to external CSS files
  • <script> - Link to JavaScript files

HTML & Accessibility 🎯

Accessible HTML: Code that works for everyone, including people with disabilities.

Key Accessibility Practices:

  • Alt text on images - Describe images for screen readers
  • Semantic HTML - Use proper headings, landmarks
  • Form labels - Connect labels to inputs with 'for' attribute
  • Color contrast - Ensure readable text colors
  • Keyboard navigation - All features accessible via keyboard
<!-- ACCESSIBLE FORM --> <label for="email-input">Email Address:</label> <input type="email" id="email-input" required>

💼 Part 9: Real-World Scenarios

Practical applications of HTML structure

🏪📰🏫

Real-World Example: Blog Post

How a blog website is structured:

<article> <header> <h1>How to Learn HTML</h1> <p>By John Smith | January 15, 2025</p> </header> <img src="html-hero.jpg" alt="HTML Code"> <p>HTML is the foundation of every website...</p> <h2>Getting Started</h2> <ol> <li>Learn basic tags</li> <li>Practice writing HTML</li> <li>Build a project</li> </ol> <footer> <p>Tags: HTML, Web Development, Learning</p> </footer> </article>

Real-World Example: E-commerce Product

Structure of an online store product page:

<article class="product"> <h1>Wireless Headphones</h1> <img src="headphones.jpg" alt="Product image"> <p>Price: <strong>$99.99</strong></p> <h2>Features</h2> <ul> <li>Bluetooth 5.0</li> <li>30-hour battery</li> <li>Noise cancelling</li> </ul> <form> <label for="quantity">Quantity:</label> <input type="number" id="quantity"> <button>Add to Cart</button> </form> </article>

Real-World Example: Student Portal

University student portal structure:

<main> <section> <h2>My Grades</h2> <table> <tr> <th>Module</th> <th>Grade</th> </tr> <tr> <td>HTML & CSS</td> <td>A</td> </tr> </table> </section> <section> <h2>Assignment Submission</h2> <form> <input type="file" accept=".pdf,.doc,.docx"> <button type="submit">Submit</button> </form> </section> </main>

❌ Common HTML Mistakes

Mistake 1: Forgetting Closing Tags

<!-- WRONG --> <p>This paragraph never closes <p>Another paragraph <!-- CORRECT --> <p>This paragraph closes</p> <p>Another paragraph</p>

Mistake 2: Nesting Issues

<!-- WRONG: Incorrectly nested --> <p>Text <div>Div inside</p></div> <!-- CORRECT --> <div><p>Text inside div</p></div>

More Common Mistakes

Mistake 3: Missing Alt Text

<!-- WRONG: No alt text --> <img src="photo.jpg"> <!-- CORRECT: Descriptive alt text --> <img src="photo.jpg" alt="A family of ducks">

Mistake 4: Wrong Attribute Syntax

<!-- WRONG: Single quotes and uppercase --> <A HREF='page.html'>Link</A> <!-- CORRECT: Lowercase and double quotes --> <a href="page.html">Link</a>

HTML5 DOCTYPE

DOCTYPE Declaration: Tells the browser this is HTML5 and not an older version of HTML.

Always Start With:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My Page</title> </head> <body> ...content... </body> </html>

Why It Matters:

  • HTML5 is current - Latest standard
  • Simpler - Much shorter than older declarations
  • Required - Every page must have it

Validating Your HTML 🔍

HTML Validation: Checking code for errors and compliance with HTML standards.

How to Validate:

  • Use W3C Validator - validator.w3.org
  • Check for closing tags - Every element properly closed
  • Validate nesting - Correct parent-child relationships
  • Browser DevTools - Right-click → Inspect Element

What the Validator Checks:

  • Missing closing tags
  • Invalid nesting
  • Unknown attributes
  • Improper DOCTYPE
  • Character encoding issues
